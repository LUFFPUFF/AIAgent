# 🚀 AI Orchestration Engine

Добро пожаловать в нашу систему для оркестрации и управления сложными AI-потоками работы. Этот проект представляет собой фреймворк, разработанный для создания автономных, многоступенчатых агентов, способных взаимодействовать с файловой системой, терминалом, и внешними ресурсами.

## ✨ Особенности проекта

* **Автономная оркестрация:** Управление рабочим процессом через центральный `Orchestrator`.
* **Модульная архитектура:** Четкое разделение на компоненты: `Provider` (источники данных), `Tools` (внешние функции), `Prompts` (конфигурация промптов).
* **Интеграция с ОС:** Встроенные инструменты для работы с файловой системой (`file_system`), Git и терминалом.
* **Конфигурируемость:** Легкое изменение логики и настроек через директорию `config/`.

## 💡 Инструкция по использованию

### 1. Установка и Зависимости

Поскольку это фреймворк, предполагается, что основные зависимости уже настроены. Если понадобятся внешние библиотеки, установите их следующим образом:

```bash
pip install -r requirements.txt
```
*(Примечание: файл `requirements.txt` должен быть создан)*

### 2. Запуск агента

Для запуска основных функций системы используйте главный скрипт `main.py`.

**Пример базового запуска:**

```bash
python main.py
```

**Пример запуска с указанием задачи (Job ID):**

```bash
python main.py --job-id "process_document_workflow" --input-file ./data/report.pdf
```

### 3. Архитектура компонентов

* **`core/orchestrator.py`:** Ядро системы. Координирует последовательность действий, запрашивает нужные инструменты и управляет состоянием агента.
* **`core/tools.py`:** Объектный класс, предоставляющий доступ к инструментам (например, `file_system.read()`, `terminal.execute()`).
* **`tools/`:** Директория с конкретными инструментами. Каждый модуль (например, `editor.py`, `git_agent.py`) реализует атомарную, тестируемую функцию.
* **`config/prompts.py`:** Центральное место для хранения и управления шаблонами промптов, обеспечивая высокую гибкость системы.

## 🛠️ Модульные инструменты (Tools)

| Инструмент | Модуль | Описание |
| :--- | :--- | :--- |
| **Файловый менеджер** | `tools/file_system.py` | Чтение, запись, поиск и манипуляции с файлами. |
| **Терминал** | `tools/terminal.py` | Выполнение произвольных команд ОС для интеграции с внешними программами. |
| **Git** | `tools/git_agent.py` | Управление состоянием репозитория (commit, status, pull). |
| **Редактор** | `tools/editor.py` | Инструмент для структурного редактирования кода или текста. |

## 📚 Разработка

Если вы хотите расширить функциональность системы:
1.  Добавьте новый инструмент в директорию `tools/`.
2.  Определите схему для этого инструмента в `schemas/chat.py` или создайте новую.
3.  Обновите логику оркестрации в `core/orchestrator.py` для использования нового инструмента.
